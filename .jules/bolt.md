## 2024-05-24 - Event Listeners & Async Form Submissions
**Learning:** Incorrectly referencing DOM IDs (e.g., `feedbackForm` instead of `upgradeForm`) breaks JS-based form handlers like AJAX submissions, silently falling back to default synchronous form actions that cause a slow full-page reload and redirection. Furthermore, duplicate globally-scoped script blocks redundantly attach duplicate event listeners, adding unnecessary JS execution and memory overhead. Also, attaching individual event listeners to multiple input fields is less efficient than attaching a single delegated listener to the parent form element.
**Action:** Always verify that JavaScript variables strictly map to their correct HTML DOM IDs to prevent fallback to slow, default browser behaviors. Consolidate and remove duplicated legacy code blocks to minimize JS parse and execution time, and prefer event delegation on parent containers over mapping individual element listeners to dramatically lower active DOM listener counts.
